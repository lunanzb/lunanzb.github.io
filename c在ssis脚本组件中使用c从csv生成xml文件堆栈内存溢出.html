c在ssis脚本组件中使用c从csv生成xml文件堆栈内存溢出

发表于 2020-08-31 |

原文标题:

to generate a xml file from csv ussing c# in ssis script component

问题:

我有一个汽车保险记录的csv文件数据，我需要使用ssis将其转换为xml文件。 现在，当我在csv中具有唯一的保险号时，可以生成所需格式的xml文件，但是会多次出现保险号。 对于前。 一个家庭的汽车保险中有4位驾驶员和4辆汽车，因此csv文件中有该保险号的4个条目，应为xml中的1块，其中保险号出现1次，所有驾驶员和车辆条目在1个保险号下出现4次标签。 没有重复的驾驶员或车辆或保险号码时，我已经生成了一个xml文件。 我对SSIS还是很陌生，从来没有用c＃编写过代码，所以如果有人可以帮助我编写代码。 如果列重复，如何遍历列并为其创建子节点。 这是我在ssis脚本组件中用于生成我需要的xml结构的代码，但仅在没有重新定位的情况下才起作用。

解决方案:

===============>>#1 票数：0

您可以在需要XML文档时将CSV读取为对象，并将构造的对象转换为XElement 。 例： 在示例中，我们创建Submission和Driver对象，其中提交可以具有一个或多个PolicyDrivers 。 每个提交都存储在字典中，其中的键是SubmissionID因此，如果单个SubmissionID有多个条目， PolicyDrivers将合并在一起。 这意味着在SubmissionID发生冲突的情况下，只有PolicyDrivers会更改。
