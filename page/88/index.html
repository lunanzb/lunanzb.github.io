cC检测光标是否被隐藏堆栈内存溢出

发表于 2020-08-31

原文标题:

C# Detecting whether cursor is hidden

问题:

是的，我的确意识到有很多这样的问题，但是没有一个对我有用。 我需要检查系统光标是否被隐藏。 我已经尝试过Cursor.Current == null 。 它什么也没做。 （我用全屏的Youtube，Discord和Krita隐藏了光标）。 我还尝试使用user32.dll中的GetCursorInfo并检查标志是否为0（隐藏）。 没有。 我的目标是在隐藏光标时隐藏屏幕覆盖UI。 那么，如何检查系统光标是否被其他方式隐藏？ 编辑： 好的，我借助艾哈迈德·阿卜杜勒·哈密德（Ahmed Abdelhameed）进行了发现。 基本上，浏览器等不会渲染光标，但是它仍然被系统视为可见。 但是，手柄已更改。 默认情况下，系统提供的句柄数量有限（ Cursors..Handle ），并且不可见的浏览器光标都不使用它们。 但是，这并不是一个完整的解决方案，因为浏览器和其他软件可以创建除看不见之外的新型游标（例如VS中向后倾斜），该游标也将返回CursorTypes.Other 。 正如Herohtar所指出的那样，隐藏的游标通常具有较大的指针值（在我的情况下， Math.Abs( pointer ) > 10000000在大多数情况下有效）。 涵盖了大多数浏览器。 老实说，我认为在检测隐藏的游标方面我们不能做更多的事情。 在极少数情况下，没有隐藏光标，但我的应用程序认为它是相反的，我决定只提供一个选项，让用户单击设置中的按钮，在经过一段时间后将当前光标添加到例外情况。

解决方案:

===============>>#1 票数：0

检测光标是否被隐藏的方法不只一种。 在某些时候，系统将被告知不要渲染它，在某些时候，软件本身将不会渲染它，有时它会以非光标的形式来渲染它（例如，数字艺术画笔轮廓）。 在最常见的情况下，当系统本身不呈现光标时，您要做的就是检查cursors标志是否为0（隐藏）。 （或执行快捷方式检查Cursor.Current == null ） 现在我们遇到了网络浏览器。 浏览器不会渲染隐藏的光标，但是会更改光标的句柄。 大多数情况下，但不一定总是句柄的值会很大。 现在，我们差不多完成了，但是不要忘记会有例外。 例如，Krita在隐藏光标时会渲染笔刷的轮廓，但是就系统而言，光标仍然可见。 没有一种简单，通用的方法可以解决此问题，因此我们几乎必须明确地告诉我们的软件以注意这些特殊情况。 在某些情况下，指针X的含义是A代表软件B，而C代表软件D。这不太可能，但是如果您想更加安全，则例外列表应该是应用程序和句柄组合的列表。
