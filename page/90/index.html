cFILEvsHANDLE这两件事有什么区别堆栈内存溢出

发表于 2020-08-31

原文标题:

FILE vs HANDLE - what is the difference between these two things

问题:

因此，我在Windows上运行，需要弄清楚文件的大小。 GetFileSize在很多地方都提到了GetFileSize ，这需要一个HANDLE 。 我已经习惯了只是工作FILE期从返回fopen和这样。

解决方案:

===============>>#1 票数：3

FILE是标准C库的一种，专用于使用stdio.h函数进行文件处理。 它不能用于其他任何用途。 C标准库具有可移植的优点。 HANDLE是Windows API使用的类似但更广泛的概念，用于他们希望在原始指针之上添加抽象层的所有内容。 它用于文件，端口，线程，互斥体，事件，管道等。 通常：您可以在其上使用WaitForSingleObject任何Windows资源WaitFor...函数是Window使线程有效休眠而不浪费CPU资源的方式。 请参阅什么是Windows句柄？ 。 它仅适用于Windows API特定功能，例如CreateFile 。 这两种都是“不透明类型”，应用程序程序员不知道或不在乎它们是如何在内部实现的。 实际上，用于Windows的标准C库端口只是Windows API之上的抽象层。 因此FILE* fp = fopen("", "w"); 将很可能归结为内部的CreateFile调用。 因此，就我们所知， FILE内部可能包含一个HANDLE 。

===============>>#2 票数：1

带有HANDLE的GetFileSize()函数是Windows API的一部分，这意味着它仅可用于Windows程序。 fopen()和FILE是任何标准C编译器随附的C运行时库的一部分，这意味着使用它们应该与操作系统无关。 您不能将FILE传递给GetFileSize()函数，因为HANDLE和FILE是两个不同的事物。
