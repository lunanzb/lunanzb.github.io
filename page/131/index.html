解决方案:

===============>>#1 票数：13 已采纳

通常 （ 正常 ）仅用于读取的集合是“非官方的”线程安全的（我知道.NET中没有集合会在读取过程中对其进行修改）。 有一些警告： 一个使用备注/延迟加载/无论您想调用它的类的简单示例，都可能破坏线程安全性。

===============>>#2 票数：1

假设您要提前加载值，则可以使用System.Collections.Immutable库中的ImmutableHashSet 。 不可变的集合将自己宣传为线程安全的 ，因此我们不必担心HashSet的“非官方”线程安全。 ...

===============>>#3 票数：1

来自Microsoft： 线程安全集合 .NET Framework 4引入了System.Collections.Concurrent命名空间，该命名空间包括几个线程安全和可伸缩的集合类。 多个线程可以安全有效地从这些集合中添加或删除项目，而无需在用户代码中进行额外的同步。 在编写新代码时，只要有多个线程将并发写入集合，就使用并发集合类。 如果仅从共享集合中读取，则可以使用System.Collections.Generic命名空间中的类。 我们建议您不要使用1.0集合类，除非您需要定位.NET Framework 1.1或更早版本的运行时。 由于Contains不会修改集合，因此它只是一个读取操作，并且由于HashSet在System.Collections.Generic ，因此并发调用Contains是绝对可以的。
