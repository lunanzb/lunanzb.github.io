原文标题:

Android Recycler View - reverse layout inside Constraint Layout

问题:

我在ConstraintLayout内有一个RecyclerView ，它在顶部受parent约束，在底部受EditText约束。 效果很好，但是，一旦我将回收站设置为reverseLayout=true ，如果我的项目比屏幕显示的要多，则必须向下滚动底部的第一项才能完全看到。 预览正是在真实设备上发生的，这让我觉得我缺少布局中的某些内容。 注意： 如果我将RecyclerView EditText的底部约束从app:layout_constraintBottom_toTopOf="@id/feedMessageInput"到app:layout_constraintBottom_toBottomOf="parent"则不会发生此问题，但是这不是可取的。 Xml布局： 多行滚动到顶部和底部时当前不正确的预览-这是不正确的： 无论从顶部滚动多少行，我如何期望reverseLayout=true起作用： 我可能会错过的任何帮助将不胜感激。

解决方案:

===============>>#1 票数：1 已采纳

根据ConstraintLayout的文档 ： 重要提示：不建议对ConstraintLayout中包含的窗口小部件使用MATCH_PARENT。 可以通过使用MATCH_CONSTRAINT来定义类似的行为，并将相应的左/右或上/下约束设置为“父”。 为什么要做这种工作？ 我的猜测是，它实际上不受支持，但尚未禁用，可能未经过测试。 简短的回答是使用match_constraints （ 0dp专），避免match_parent在任何时候，当谈到ConstraintLayout。
