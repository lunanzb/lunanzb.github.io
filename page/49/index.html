发表于 2020-08-31

原文标题:

How can I make sure there is no floating point arithmetic in a C code (Visual Studio 2013 express + WDK 8.1, WDM kernel driver)?

问题:

几年前，我遇到了一个改进的moufiltr驱动程序（由Povohat制造），该驱动程序允许用户设置自定义的鼠标加速。 该驱动程序使用浮点算法。 我在WDK7样本中找到了一个mouclass样本驱动程序，基于povohat的moufiltr驱动程序，我已经修改了该样本以包括可调加速度（来自Quake 3 cl_input.c的等式），但是使用了升序的整数而不是双精度整数，因此不需要KeSaveFloatingPointState。 这样做的目的是也为Windows XP提供一些支持。 如果在计算中只使用类型LONGLONG变量（+，/，*，％），则可以假设没有使用浮点算法吗？

解决方案:

===============>>#1 票数：0

即使您的代码没有浮点文字，浮点变量和浮点强制转换，也不能保证在编译的机器代码级别没有浮点运算。 通常，在关闭优化的情况下，编译后的代码基本上是源代码的直接翻译，在这种情况下，您的机会更大，如果源代码中没有浮点机制，则编译后的代码都不会。 但是，编译器通常具有对目标体系结构的指令集的完全访问权限，并且在启用优化的情况下，它们将根据优化标准（通常是速度和/或大小）对最终的编译输出进行大量修改。 在这种情况下，如果编译器认为必要的某些（或全部）浮点指令，则即使您的源代码中没有这些浮点指令，它们也将出现在编译的机器代码中。 如果您的编译器不支持限制可以使用的机器代码（汇编器）指令，而VS 2013 Express不支持，则唯一的发现方法是对已编译的代码进行反编译并自行查找浮点指令。
