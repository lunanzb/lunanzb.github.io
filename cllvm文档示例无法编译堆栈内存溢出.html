解决方案:

如您所知，std :: make_unique至少需要c ++ 14，而llvm-config扩展提供的-std = c ++ 11标志会覆盖您提供的标志。 作为一种快速的解决方法，您只需将-std = c ++ 17移动到llvm-config扩展的右侧： 这意味着clang将接收矛盾的参数（-std = c ++ 11和-std = c ++ 17），但是以我的经验，最右边的将是生效的。 参见： https : //godbolt.org/z/xwujes
