aspnet如何处理每个请求的会话AspnetMVC堆栈内存溢出

发表于 2020-08-31 |

原文标题:

How to handle Session on each request Asp.net mvc

问题:

我有一个运行在移动应用程序（WebView）中的Web应用程序。当用户打开该应用程序时，移动应用程序会通过请求向我发送电话的语言信息。 由于语言值仅在第一个请求中出现，然后才被删除，因此我必须将其保存起来并存储以用于后续请求。 我已经在global.asax上尝试过PreRequestHandlerExecute和Application_AcquireRequestState。 我控制会议。 如果为null，则从请求中获取值；如果不为null，则从会话中获取值。 但是似乎session总是为null，并且总是从请求中获取值。 有什么想法会很有帮助吗？ 感谢前进。

解决方案:

===============>>#1 票数：0

按照您的代码，您总是会进入第一个if语句，该语句将读取并设置代码形式dilKodu 您的代码应更像这样 基本上，您要执行的操作的if语句设置不正确。 另外，建议您保持会话对象较小，即仅将langName存储在会话中，然后每次从该对象中创建一个CultureInfo，而不是存储CultureInfo。

===============>>#2 票数：0

这是MVC网络应用吗？ 如果是这样，您可以尝试将值保存在Controller.TempData 属性中 。 因此，而不是执行this.Session["culture"] = ci; 您将添加到TempData属性TempData.Add("culture", ci); 。 这将使文化可用于下一个请求。 您只需要确保在每个请求中重新添加ci值即可使该值持续存在您需要的时间。 TempDataDictionary的文档： https ://docs.microsoft.com/zh-cn/dotnet/api/system.web.mvc.tempdatadictionary ? view = aspnet-mvc-5.2

===============>>#3 票数：0

这是因为在web.config中进行了设置。 我试图保护cookie，但它阻止了会话。 我删除了它，现在一切正常。 但是我仍然需要找到一种保护cookie的方法。
